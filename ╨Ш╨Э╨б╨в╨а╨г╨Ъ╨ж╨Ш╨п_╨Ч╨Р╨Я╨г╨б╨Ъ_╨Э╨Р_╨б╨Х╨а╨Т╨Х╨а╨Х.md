# Как запустить VisAsia CRM в интернете (Railway + Turso)

Это пошаговая инструкция для первого раза. Не торопись, делай по порядку.
Тебе понадобится: компьютер, почта, телефон (для подтверждений). Время: ~30–40 минут.

Вся система будет состоять из двух бесплатных частей:
- **Turso** — облачная база данных (где хранятся клиенты, сделки, оплаты).
- **Railway** — где работает сам сайт CRM.

---

## ЧАСТЬ 1. База данных в облаке (Turso)

### Шаг 1.1 — Регистрация
1. Зайди на сайт **turso.tech**
2. Нажми **Sign Up** (Регистрация). Проще всего войти через GitHub или Google.
3. Подтверди почту, если попросят.

### Шаг 1.2 — Создать базу
1. На главном экране (Dashboard) нажми **Create Database** (Создать базу).
2. Имя базы: напиши `visasia` (латиницей).
3. Регион (Location): выбери ближайший — например **Frankfurt** (Европа) или любой ближайший к Казахстану.
4. Нажми **Create**.

### Шаг 1.3 — Получить адрес базы и ключ
Тебе нужны ДВЕ строки. Сохрани их в блокнот — они понадобятся в Части 2.

1. Открой созданную базу `visasia`.
2. Найди **Database URL** (адрес базы) — он начинается с `libsql://...`. Скопируй.
3. Найди кнопку **Create Token** (или «Generate Token» / «Tokens»). Нажми, создай токен.
4. Скопируй сам **токен** (длинная строка букв и цифр). ВАЖНО: его показывают один раз — сразу сохрани.

Итого у тебя должно быть:
- `TURSO_DATABASE_URL` = `libsql://visasia-...turso.io`
- `TURSO_AUTH_TOKEN` = `eyJ...` (длинная строка)

### Шаг 1.4 — Залить структуру и данные в облачную базу
Это делается один раз, чтобы в облаке появились таблицы и твои данные.
Если не уверен — этот шаг можно сделать со мной отдельно, я подскажу команды.
(Технически: установить Turso CLI и выполнить импорт, либо я подготовлю отдельный скрипт.)

## ЧАСТЬ 2. Сайт CRM на Railway

### Шаг 2.1 — Регистрация
1. Зайди на **railway.app**
2. Нажми **Login** → войди через **GitHub** (проще всего; если нет GitHub — сначала заведи бесплатный аккаунт на github.com).
3. Railway может попросить привязать карту для подтверждения — это нормально, оплата только по факту использования (несколько $ в месяц).

### Шаг 2.2 — Загрузить код проекта
Вариант, который проще для первого раза:
1. Залей папку проекта VisAsia в репозиторий на **GitHub** (я подскажу как, если нужно).
2. На Railway нажми **New Project** → **Deploy from GitHub repo** → выбери свой репозиторий.

Railway сам поймёт, что это Next.js, и начнёт сборку.

### Шаг 2.3 — Вписать переменные (САМОЕ ВАЖНОЕ)
Это «настройки», которые связывают CRM с базой и защищают вход.
1. В проекте на Railway открой вкладку **Variables** (Переменные).
2. Добавь по очереди (кнопка **New Variable**):

```
TURSO_DATABASE_URL = (адрес из шага 1.3)
TURSO_AUTH_TOKEN   = (токен из шага 1.3)
AUTH_SECRET        = (придумай длинную случайную строку, например 30+ символов)
NODE_ENV           = production
```

`AUTH_SECRET` — это секретный ключ для защиты входа в систему. Просто набери длинную случайную строку (буквы+цифры) и никому не показывай.

3. Сохрани. Railway автоматически пересоберёт проект с новыми настройками.

### Шаг 2.4 — Получить адрес сайта
1. В проекте открой **Settings** → раздел **Networking** (или **Domains**).
2. Нажми **Generate Domain**.
3. Получишь адрес вида `visasia-production.up.railway.app` — это и есть ссылка на твою CRM в интернете.

### Шаг 2.5 — Первый вход
1. Открой полученный адрес в браузере.
2. Войди под своим логином и паролем (теми, что в базе).
3. При первом входе старый пароль автоматически защитится (захэшируется) — это правильно, так и задумано.

---

## ЧТО ИЗМЕНИЛОСЬ В КОДЕ (для понимания)

- CRM теперь умеет работать с облачной базой (Turso). Локально (у тебя на компьютере) всё работает как раньше — если переменные Turso не заданы, используется локальный файл.
- Пароли больше не хранятся в открытом виде — они защищены (хэшируются). Старые пароли продолжают работать и защитятся при первом входе.
- В карточку клиента добавлены поля для Telegram (`telegramChatId`, `botStarted`) — это задел под приём заявок и рассылки через бота.

---

## ВАЖНО / ЧАСТЫЕ ВОПРОСЫ

**Сломается ли локальная версия?** Нет. На твоём компьютере без переменных Turso CRM работает с локальной базой как прежде.

**Это точно бесплатно?** Turso — бесплатно (с большим запасом). Railway — по потреблению, обычно несколько долларов в месяц для небольшой команды.

**Что делать, если что-то не получилось?** Не паникуй, сделай скриншот ошибки — разберём вместе по шагам. Самое частое: опечатка в переменных (Часть 2, шаг 2.3) — проверь, что адрес и токен скопированы полностью.

**Следующий шаг после запуска:** когда CRM заработает в интернете — беремся за Telegram-бота (приём заявок), ради чего всё и затевалось.
